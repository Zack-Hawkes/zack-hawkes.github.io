Buggin' Out

About the Project

Buggin' Out is an online multiplayer, bug-catching game where you interact, compete and care for bugs. The game is still currently in development, but we are working on it as part of a third year university assignment.

Multiplayer Networking and My Role

This game is my first *proper* online multiplayer project. I have limited experience when working with networking through various other old projects, however it was this project that first introduced me to subjects such as server authority, peer to peer networking and the use of steam API.

As one of two programmers on the team, I have been mostly focused on the networking and multiplayer aspects of the game. I developed the AI system for the bugs, a fully networked and modular parent script for minigames and the majority of the player movement and inventory mechanics.

A key focus when working on this project has been server authority. Since the game does feature competitive elements, cheating could play a large role if / when we go public with the game. To combat this I developed the core components of the game with multiplayer in mind. The server maintains an authoritative list of all bugs other players own, including their stats, rarity and customisations. Whilst the player can easily modify stats locally, and mofidications to stats they make are purely visual until validated by the server.

All core components follow this same structure, bug catches are verified server side, checking player proximity before confirming the catch, minigame results are calculated server side to prevent manipulating results and all inventory changes and shop purchases are verified by the server before being applied.

Shader Development

Another key role I played in the game, was developing custom shaders to enhance the visual appeal of the game, and to improve the workflow within the Unity editor.

One of the shader's I am most proud of is the terrain shader, using triplanar mapping and normal-based colouring. This allows us to carve mountains and valleys seamlessly without texture stretching or the need to manually paint cliff faces. I also set this up to work alongside unity's built it texture system. By painting on new textures we can create different biomes, with their own cliff and floor colours, greatly improving the speed of which we develop the map.

Another shader I developed was one for the skybox. Using this shader, we can blend multiple colours to create gradients and a dynamic day and night cycle. I made use of multiple noise techniques, including simplex and voronoi. At night I used these to create stars that slowly twinkle and rotate. And I used a similair method to create stylised clouds in the sky, that pan across the map casting interesting shadows on the floor.
